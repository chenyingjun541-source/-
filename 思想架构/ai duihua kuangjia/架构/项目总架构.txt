# 项目总架构

## 一、核心架构理念

**本地主权，云端代工**

- 所有核心数据存储在本地，用户完全掌控
- 云端只做临时处理，处理后立即销毁数据
- 本地小模型快速响应，云端大模型深度处理

## 四、人格权重机制 🎭

```json
{
  "base_traits": ["热情", "幽默", "亲切", "谨慎", "讨厌"],
  "dynamic_traits": ["可对话中生成的新性格"],
  "metrics": {
    "intimacy": { "range": "0-10", "update": "每次互动" },
    "trust_level": { "range": "0-10", "update": "每个里程碑" }
  }
}
```

**规则**：
- 底层性格：-10 到 10，可为负
- 动态性格：不能为负，为负时扣除底层性格
- 亲密度：每次互动更新
- 信任度：每个里程碑更新

## 五、分层模型策略 🤖

| 模型 | 定位 | 任务 | 代表 |
|------|------|------|------|
| 本地小模型 | 潜意识/快速响应 | 简单问候、脱敏润色、性格注入 | Qwen-1.8B-Int4 |
| 云端快速 | 意识/快速思考 | 日常对话、情感回应 | 阿里云百炼 |
| 云端深度 | 意识/深度思考 | 记忆提炼、复杂推理、长篇生成 | DeepSeek |

**路由决策树**：
```
用户输入
    │
    ├── 关键词命中 ──→ 快速路由
    │
    ├── 学习词命中 ──→ 已学习路由
    │
    ├── 本地模型判断(3秒) ──→ 命中 ──→ 本地处理
    │
    └── 未命中 ──→ 云端快速处理
                    │
                    └── 复杂推理 ──→ 云端深度处理
```

|------|------|------|----------|

## 八、与airi的集成关系

```
本项目 (houduan)
    │
    ├── hot-swap.ts  ←── 代理模式时被加载的"插槽"
    │
    ├── local.model/ ←── 智能路由、本地模型调用
    │
    └── proxy/       ←── HTTP/WebSocket透明代理

airi 项目 (前端+后端)
    │
    ├── 前端 (stage-web) ←── 用户界面
    │
    └── 后端 (server) ←── Docker容器内运行，6112端口
```

**重要**：
- houduan 是你的**实际开发项目**，一直在更改和添加
- airi 是**参考/集成的上游项目**，通过代理模式接入
- 通过 `interface-config.json` 确定接口，通过 `hot-swap.ts` 接入插槽

## 九、关键设计原则

1. **背景文件要精简** - 防止拖慢本地模型，<1MB
2. **本地提示词<60字** - 云端加工，通过hot-swap实时更新
3. **人格分即时/普通** - 即时人格用于快速响应场景
4. **输入层面备注** - 记忆注入是给模型输入加备注，不是干预输出
5. **云端零存储** - 处理后强制销毁数据
